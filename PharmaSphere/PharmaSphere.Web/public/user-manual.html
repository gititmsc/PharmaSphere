1. System Overview

PharmaSphere is a web-based Sales Order Management System designed for pharmaceutical manufacturing teams. It digitises and tracks the complete lifecycle of a sales order — from initial creation by the Admin through QA approval, artwork, packaging material supply, production, packing, and final dispatch.

The system enforces a strict role-based workflow, ensuring that each department can only act on orders at their specific stage. This eliminates miscommunication, provides full audit traceability, and gives management a real-time pipeline view.

Key Capabilities

  • Role-gated sales order creation, editing, and status progression
  • Automatic status advancement when required fields are saved
  • Real-time dashboard with pipeline counts per stage
  • Complete audit log and status history per order
  • Secure JWT authentication with optional Two-Factor Authentication (2FA)
  • Admin-only access to sensitive fields (Rate, Amount) and management features
  • Dedicated QA / Design tab for regulatory permission dates (Product Permission, COPP, FSC)

Supported Browsers

Google Chrome 110+, Microsoft Edge 110+, Mozilla Firefox 110+, Safari 16+. The application is fully responsive and works on mobile devices.

Application URL

Access the application at your organisation's configured URL. Contact your system administrator for the correct address.

3. Authentication

3.1 Login

  1. Navigate to the application URL. The Login screen appears.

  2. Enter your Email Address and Password.

  3. Click Sign In.

  4. On success you are redirected to the Dashboard.

Login failed? Check your email and password. After multiple failed attempts the account may be locked — contact your Admin.

3.2 Two-Factor Authentication (2FA)

If 2FA is enabled on your account:

  1. After entering credentials you are redirected to the Verify 2FA screen.

  2. Check your registered email for a one-time code.

  3. Enter the code and click Verify.

  4. On success you are redirected to the Dashboard.

Note The 2FA code expires after a short period. If it has expired, restart the login process to receive a new code.

3.3 Forgot Password

  1. On the Login screen click Forgot Password?

  2. Enter your registered email address and click Send Reset Link.

  3. Open the email and click the reset link (valid for a limited time).

  4. Enter and confirm your new password, then click Reset Password.

  5. Log in with your new password.

3.4 Logout

  1. Click your Avatar in the top-right corner of any screen.

  2. Click Logout from the dropdown menu.

4. Dashboard

The Dashboard is the home screen after login. Its content varies by role.

4.1 Admin Dashboard

Admins see four sections:

Summary Cards

  • Total Orders — all-time order count
  • Active Orders — orders currently in progress (not Dispatched or Cancelled)
  • Dispatched — completed orders
  • Cancelled — cancelled orders

Workflow Pipeline

Six colour-coded cards show the live count of orders at each stage. Click any card to jump to the Sales Order list filtered by that status.

Recent Orders

The last 10 orders across all statuses with a direct link to edit each one.

4.2 Role-Specific Dashboard

Non-admin users see a personalised work queue:

  • A highlighted count card showing how many orders are waiting for their action (orange = work pending, green = all clear).
  • A Go to My Orders button that navigates to the filtered Sales Order list.
  • A work queue table listing every order at their stage with a direct edit link per row.
Tip The Dashboard refreshes its data every time you navigate back to it. Use the sidebar Dashboard link or the logo to refresh.

5. Sales Orders

5.1 Sales Order List

Navigate to Sales Orders in the left sidebar. The list shows orders filtered by your role automatically.

Filtering (Admin only)

  • Search — type any part of the Order No, Party, or Brand Name.
  • Status — filter by a specific workflow status.
  • From / To Date — filter by Order Date range.
  • Click the red filter icon to clear all active filters.

Column Visibility

Click the columns icon (grid icon) to show or hide columns. The Amount column is only available to Admin users.

Row Actions

  • Edit (pencil icon) — opens the order form for editing.
  • Delete (trash icon) — soft-deletes the order. Admin only.

5.2 Adding a New Sales Order (Admin Only)

  1. Click + Add Order in the top-right of the Sales Orders list.

  2. Fill in the General Info tab — Order No, Order Date, Brand Name, Party, Generic Name, Qty, Shelf Life, MRP, Make, Neutral Code / Mfg. Lic. No / LL NO, and Admin Remarks are on this tab. Rate (Admin-only) auto-calculates Amount = Rate × Qty.

  3. Optionally fill the Packaging Material tab with component details (Vial, Label, Mono Box, Month Box, Tray, Shipper, Hologram, etc.).

  4. Click Create Order. The order is created with status PIS Pending.

Brand Name Auto-fill When you select an existing brand name, the form pre-fills details from the most recent order for that brand to save data entry time.

5.3 Editing a Sales Order

  1. Click the Edit icon on any row in the list, or click a row in your Dashboard work queue.

  2. The current order status is shown as a colour-coded chip in the top-right of the page header.

  3. Edit only the fields available to your role (other fields are read-only).

  4. Click Save Changes. If your entry satisfies the stage criteria the status advances automatically.

5.6 Cancelling a Sales Order (Admin Only)

  1. Open the sales order in edit mode.

  2. Click the red Cancel Sales Order button in the footer.

  3. A confirmation dialog appears. Confirm to proceed.

  4. The order status changes to Cancelled and becomes fully read-only.

This action cannot be undone through the UI. The Cancel button is hidden once an order is Dispatched or already Cancelled.

5.7 Deleting a Sales Order (Admin Only)

  1. In the Sales Order list, click the delete (trash) icon on the target row.

  2. A confirmation dialog appears with the order number.

  3. Confirm to soft-delete. The order is removed from the list.

Note Deletion is a soft-delete — the record is retained in the database and can be restored by a system administrator if needed.

6. Role-Specific Guides

6.1 QA Role — QA / Design Approval

You are responsible for completing all regulatory design approvals before artwork can begin. This requires filling 17 date fields on the QA / Design tab.

  1. Log in. Your Dashboard shows all orders in PIS Pending status.

  2. Click an order from the work queue or navigate to Sales Orders.

  3. Go to the QA / Design tab. An info banner reminds you that all fields must be completed to advance the status.

  4. In the QA Approval section, enter the PIS Approval Date.

  5. In the Product Permission section, enter the Apply Date, Draft Date, Approval Date, and Received Date.

  6. In the Combipack Product Permission section, enter the same four dates.

  7. In the COPP section, enter the same four dates.

  8. In the FSC section, enter the same four dates.

  9. Click Save Changes. All 17 fields must be filled for the status to automatically advance to Artwork Pending. Saving with any field empty saves the record but keeps the status at PIS Pending.

  10. Once the status advances, the order disappears from your queue.

All 17 fields are required. PIS Approval Date alone is not sufficient. Every Apply, Draft, Approval, and Received date across all four sections (Product Permission, Combipack Product Permission, COPP, FSC) must be saved before the status advances.
Access restriction Once an order moves to Artwork Pending, it is read-only for your role.

6.2 Designer Role — Artwork Approval

You sign off on the artwork before packaging materials are ordered.

  1. Log in. Your Dashboard shows all Artwork Pending orders.

  2. Open an order and go to the QA / Design tab.

  3. An info banner will confirm you can enter the Artwork Approval Date.

  4. Enter the Artwork Approval Date.

  5. Click Save Changes. Status advances to PM Supply Pending.

6.3 PPMC Role — Packing Material Supply

You coordinate ordering and receipt of all 10 packaging material components.

  1. Log in. Your Dashboard shows all PM Supply Pending orders.

  2. Open an order and go to the Packaging Material tab.

  3. Fill in all 5 Supply Vendor Approval Dates (Mono Box, Label, Insert, Tray, Shipper) under the Packing Material Order section.

  4. Fill in all 5 Production Dates (Mono Box, Label, Insert, Tray, Shipper) under the Packing Material Receive section.

  5. Click Save Changes. All 10 fields must be filled for the status to advance to Production Pending.

All 10 fields are required. Saving with any field empty will save the record but will not advance the status. Fill all fields and save again.

6.4 Production Role — Filling Plan

You enter the filling production schedule.

  1. Log in. Your Dashboard shows all Production Pending orders.

  2. Open an order and go to the Production Info tab.

  3. Enter the Filling Plan Date (and optionally Sterility 14 Days Date).

  4. Click Save Changes. Status advances to Packing Pending.

6.5 Packing Role — Packing Plan

You schedule the packing operation after filling is complete.

  1. Log in. Your Dashboard shows all Packing Pending orders.

  2. Open an order and go to the Production Info tab.

  3. Enter the Packing Plan Date.

  4. Click Save Changes. Status advances to Dispatch Pending.

6.6 Dispatcher Role — Dispatch

You record the actual dispatch and complete the order lifecycle.

  1. Log in. Your Dashboard shows all Dispatch Pending orders.

  2. Open an order and go to the Production Info tab.

  3. Enter the Dispatch Date.

  4. Click Save Changes. Status advances to Dispatched. The order is now complete and read-only for all non-admin users.

7. User Management (Admin Only)

The Users menu item in the sidebar is visible only to Admin users. Other roles cannot access this section.

Viewing Users

  1. Click Users in the left sidebar.

  2. The user list shows all registered users with their name, email, role, and active status.

Adding a New User

  1. Click + Add User.

  2. Fill in First Name, Last Name, Email Address, Password, and Role.

  3. Click Save. The user can now log in with the provided credentials.

Editing a User

  1. Click the Edit icon next to the user.

  2. Update the required fields (password is optional on edit).

  3. Click Save.

Available Roles

  • Admin
  • QA
  • Designer
  • PPMC
  • Production
  • Packing
  • Dispatch

9. Glossary

TermDefinition
PISProduct Information Sheet — the initial product specification document that QA must approve.
Generic NameThe non-proprietary (chemical) name of the drug product; previously labelled "Composition" in the form.
MRPMaximum Retail Price — the highest price at which the product can be sold to the end consumer.
RateThe per-unit price set by Admin. The system automatically calculates Amount = Rate × Qty. Visible and editable by Admin only.
Neutral Code / Mfg. Lic. No / LL NORegulatory reference codes associated with the product's manufacturing licence and lot/licence number.
Product Permission (PP)Regulatory permission for a single product. Tracked by Apply, Draft, Approval, and Received dates.
Combipack Product Permission (CPP)Regulatory permission for a combination-pack product. Tracked by Apply, Draft, Approval, and Received dates.
COPPCertificate of Pharmaceutical Product — an international regulatory certificate. Tracked by Apply, Draft, Approval, and Received dates.
FSCFree Sale Certificate — certifies that the product is approved for sale. Tracked by Apply, Draft, Approval, and Received dates.
QA / Design TabThe dedicated tab on the Sales Order form containing all QA regulatory dates (PIS Approval, Artwork Approval, PP, CPP, COPP, FSC). Visible to Admin and QA; Artwork date also visible to Designer.
PPMCProduction Planning & Material Control — the team responsible for ordering and tracking packaging materials.
PM SupplyPackaging Material Supply — the stage where all 10 packaging component orders and receipts are recorded.
Soft DeleteA deletion that hides the record from the UI but retains it in the database for potential recovery.
UATUser Acceptance Testing — the process of validating that the system meets business requirements before go-live.
2FATwo-Factor Authentication — a second verification step (OTP sent to email) after password entry.
JWTJSON Web Token — the secure token used to maintain authenticated sessions.
Audit LogA record of every field change made to a sales order, including who made it and when.
Status HistoryA timeline of every status change an order has passed through.
PipelineThe Admin Dashboard view showing live order counts at each workflow stage.
